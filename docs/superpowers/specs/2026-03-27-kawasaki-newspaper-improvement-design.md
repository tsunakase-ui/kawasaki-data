# 川崎こども新聞 改善設計書

**日付**: 2026-03-27
**目的**: 適性検査（川崎市立川崎高等学校附属中学校）の過去問分析に基づき、新聞の問題品質・グラフ表示・学習効果を大幅改善する

---

## 背景・課題

### 過去問分析から判明した現状のギャップ

過去問（R8〜R7年度 適性検査Ⅰ）を解析した結果、実際の検査と現在のシステムには以下のギャップがある。

| 課題 | 現状 | 過去問の要求 |
|------|------|-------------|
| グラフ | Unsplash写真のみ | 折れ線・棒・帯・円グラフを実際に読む |
| クイズ形式 | 5択1正解（消去法で解ける） | 「当てはまらないもの」「すべて選べ」「記述式」混在 |
| データ連携 | 単一数値の羅列 | 2つのデータを組み合わせて考える |
| 計算問題 | なし | 割合・比率の計算 |
| 問題とグラフの紐づけ | なし | 問題は必ず特定のグラフを参照する |

### 実際の過去問パターン（R8適性検査Ⅰ 問題1）

- 〔資料1〕東京年平均気温推移（折れ線グラフ・100年分）
- 〔資料2〕川崎市ごみ排出量＋人口（2軸グラフ）
- 〔資料3〕プラスチック分別率（折れ線グラフ）
- 〔資料4〕各資源物分別率（棒グラフ）
- 問(1): 〔資料1〕の説明として**当てはまらないもの**を選べ（5択1解）
- 問(2): 〔資料2〕〔資料3〕から読み取れることを選べ（5択1解、2グラフ合わせ読み）
- 問(3): 下線部②を説明する最も適切なグラフを選べ（4択1解）
- 問(4): 〔資料2〕〜〔資料4〕をもとに50字以内で記述

## 設計

### アーキテクチャ変更

```
現在のフロー:
テーマ選定 → スクレイピング → Gemini生成 → JSON保存 → 表示

新しいフロー:
テーマ選定 → データセット選択 → スクレイピング → Gemini生成 → JSON保存 → 表示
                ↑
         静的データセット群(data/)
```

### Section 1: 静的データセット（`data/` ディレクトリ）

#### ファイル構成

```
data/
├── population.json       人口推移・世帯数・区別人口（2010〜2024）
├── waste.json            ごみ排出量・資源化量・焼却量（2010〜2024）
├── industry.json         事業所数・製造品出荷額（1952〜2014）
├── environment.json      多摩川BOD値・気温変化（1960〜2024）
├── energy.json           再生可能エネルギー比率・発電電力量（2013〜2023）
├── traffic.json          主要駅利用者数（2015〜2023）
├── welfare.json          高齢化率・介護施設数・待機児童数（2015〜2024）
└── finance.json          歳入・歳出・市税収入（2015〜2024）
```

#### テーマとデータセットの対応

| テーマカテゴリ | 使用データセット |
|--------------|----------------|
| 人口動態・世帯・高齢化 | population.json + welfare.json |
| 産業・工業・貿易 | industry.json |
| 環境問題・公害と再生 | environment.json + waste.json + energy.json |
| 市民生活・行政・財政 | finance.json + welfare.json |
| 地理・交通 | traffic.json + population.json |

#### データ作成方針

- **実データ優先**: 川崎市統計書・川崎市オープンデータから収集
- **AI補完**: 取得困難なデータはGeminiの知識から合理的な数値を生成（「約」付き）
- **年度範囲**: 2010〜2024年（過去問でよく使われる範囲）

### Section 2: 記事JSONスキーマ拡張

既存フィールドは変更なし。`charts[]` と `exercises[]` を追加する。

```json
{
  "theme": { ... },
  "sections": {
    "topStory": { ... },
    "conceptMap": { ... },
    "dataInsight": {
      "facts": [...],
      "trend": "..."
    },
    "dialogue": { ... },
    "charts": [
      {
        "id": "chart1",
        "type": "line",

        "xLabel": "年度",
        "datasets": [
          {
            "label": "ごみ総排出量（万トン）",
            "data": [{"x": "2015", "y": 58.2}, {"x": "2016", "y": 56.8}],
            "yAxisId": "left"

            "data": [{"x": "2015", "y": 147.3}, {"x": "2016", "y": 149.1}],
            "yAxisId": "right"
          }
        ]
      }
    ],
    "exercises": [
      {
        "type": "single",
        "question": "〔グラフ1〕から読み取れないものはどれですか？",
        "chartRef": "chart1",
        "choices": [
          {"id": "A", "text": "..."},

          {"id": "C", "text": "..."},

          {"id": "E", "text": "..."}

        "correctAnswers": ["C"],
        "explanation": "..."
      },
      {
        "type": "multiple",
        "question": "〔グラフ1〕〔グラフ2〕から読み取れることをすべて選びましょう。",
        "chartRef": ["chart1", "chart2"],
        "choices": [...],
        "correctAnswers": ["A", "D"],
        "explanation": "..."
      },
      {
        "type": "essay",
        "question": "「ごみを焼却する量を減らす必要があるので、」に続くように50字以内で書きましょう。",
        "chartRef": "chart1",
        "maxLength": 50,
        "modelAnswer": "プラスチックを分別してリサイクルに出し、燃やすごみの量を減らすことができます。",
        "explanation": "..."
      }
    ]
  },
  "quiz": { ... },
  "sources": [...],
  "glossary": [...]
}
```

**exercisesの問題タイプ**:
- `single`: 5択1正解（「当てはまらないものはどれか」形式必須）
- `multiple`: 5択複数正解（「すべて選べ」形式）
- `essay`: 記述式（文字数制限付き、模範解答あり）
- `calculation`: 計算問題（将来拡張用）

**chartRef**: 文字列（単一グラフ参照）またはstring[]（複数グラフ参照）の両方を許容する。フロントエンドは配列に正規化して処理する。

### Section 3: プロンプト改善（`scripts/prompt.mjs`）

#### システムプロンプトへの追加

```
【グラフ・問題に関する絶対ルール】

1. chartsは必ず1〜2個生成し、渡された統計データの実際の数値を使うこと。
2. exercisesは3問構成とし、必ずsingle・multiple・essayを1問ずつ含めること。
3. 各問題のchartRefは必ず対応するグラフのidを設定すること。
4. 消去法で解ける問題は禁止。選択肢はすべて「一見もっともらしく」作ること。
5. singleは「当てはまらないものはどれか」形式にすること。
6. multipleは「すべて選べ」形式にすること。正解は2〜3個にすること。
7. essayの問いかけは「〇〇なので、」で書き出す形にすること。
8. explanationには「グラフのここ（具体的な数値）を見ると〜」という説明を必ず含めること。
```

#### 生成プロンプトへのデータセット注入

```javascript
// scripts/generate.mjs で selectedDataset を選択して渡す
const datasetContext = selectedDataset
  ? `\n\n## 使用する統計データ（chartsの数値はここから取ること）\n${JSON.stringify(selectedDataset, null, 2)}`
  : '';
```

### Section 4: フロントエンド（`src/article.js` + `article.html`）

#### 依存追加

```json
// package.json
"chart.js": "^4.4.0"
```

#### UIコンポーネント

**1. グラフ表示エリア**
- `dataInsight` セクションの下に `charts[]` をChart.jsで描画
- 対応チャートタイプ: `line` / `bar` / `doughnut` / `stacked-bar`
- 2軸グラフ（`yAxisId: "left"/"right"`）に対応

**2. 問題セクション（exercisesウィジェット）**

```
┌─────────────────────────────────────────┐
│ 問題1（グラフ1を見て答えましょう）         │
│                                          │
│ 〔グラフ1〕から当てはまらないものを選べ   │
│                                          │
│ ○ A: 〇〇〇〇〇〇〇〇                  │
│ ○ B: 〇〇〇〇〇〇〇〇                  │
│ ○ C: 〇〇〇〇〇〇〇〇                  │
│ ○ D: 〇〇〇〇〇〇〇〇                  │
│ ○ E: 〇〇〇〇〇〇〇〇                  │
│                                          │
│           [答えを確認する]               │
└─────────────────────────────────────────┘
```

- `multiple` の場合はチェックボックス（複数選択可）
- `essay` の場合はテキストエリア（文字数カウント付き）

**3. 答え合わせ表示**
- 「答えを確認する」押下後に結果を展開
- 正誤を各選択肢に色表示（緑＝正解・赤＝不正解）
- `explanation` を表示
- `essay` は模範解答（`modelAnswer`）を表示

## 実装順序

1. `data/` 静的データセットを作成（8ファイル）
2. `scripts/themes.mjs` にデータセット対応テーブルを追加
3. `scripts/generate.mjs` でデータセット選択・注入ロジックを追加
4. `scripts/prompt.mjs` のシステムプロンプト・生成プロンプトを書き換え
5. `package.json` に Chart.js を追加
6. `src/article.js` にグラフ描画・exercisesウィジェットを実装
7. `article.html` のレイアウトを調整
8. 手動テスト：1記事生成して問題の品質・グラフ表示を確認

## 後方互換性

- 既存の `quiz` フィールドはそのまま残す（古い記事の表示が壊れない）
- `exercises` がない場合は `quiz` にフォールバック

- GASによる配信方法の変更
- モデルのGemini以外への切り替え
- 既存記事の遡及的な再生成
